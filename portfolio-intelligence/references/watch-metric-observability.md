# 观察指标可数据化评估

更新日期：2026-05-18

## 结论

当前 `holdings.yaml` 里的观察指标可以分为三层：

| 层级 | 含义 | 可否进入自动建议 | 典型指标 |
|---|---|---|---|
| A层：硬数据 | 每日可稳定取得，能直接参与规则判断 | 可以 | 价格、涨跌幅、成交额、MA5/MA20、指数强弱、汇率、黄金/铜/原油、美债/VIX、南向/北向资金 |
| B层：半结构化数据 | 可从公告、新闻、财报、研报标题中抽取事件，但需要关键词和人工复核 | 可以作为加权信号，不单独触发交易动作 | 政策新闻、订单、BD交易、临床数据、回购、资本开支、财报毛利率 |
| C层：弱数据或需付费 | 免费源不稳定或没有标准口径，需要人工维护或付费源 | 暂不自动触发，仅做提醒 | 游戏流水、视频号商业化、客户结构、景气预期、港股利率敏感性 |

## 只能做事件信号的指标

| 指标 | 自动化方式 | 局限 |
|---|---|---|
| 政策新闻 | 按关键词抓新闻标题和摘要，如“电网投资”“医保”“出口管制”“平台经济” | 需要人工判断政策强度 |
| 特高压订单、船舶订单、储能招标 | 新闻/公告关键词命中，记录事件数量和方向 | 免费源难以完整覆盖订单金额 |
| BD交易、临床数据 | 新闻关键词，如“BD”“授权”“临床”“获批”“III期” | 需要识别利好/利空语义 |
| 回购强度 | 港股公告/新闻关键词，或后续接公告源 | 当前新闻源可粗略跟踪，精确金额需公告接口 |
| 财报毛利率、矿山产量、客户结构 | 财报接口或公告文本 | 季度更新，不适合每日波动判断 |

## 暂不建议自动化触发的指标

| 指标 | 原因 | 建议处理 |
|---|---|---|
| 游戏流水与版号 | 版号可查，流水通常来自第三方榜单/研报，免费源不稳定 | 作为新闻观察，不作为硬触发 |
| 广告与视频号商业化 | 缺少每日公开口径 | 放入财报/管理层表述跟踪 |
| AI服务器PCB需求 | 多来自产业链新闻和研报 | 用 PCB 板块、铜价、相关个股趋势做代理 |
| 景气预期 | 主观性强，源不统一 | 用行业指数趋势、成交额、新闻热度做代理 |
| 港股利率敏感性 | 需要模型化，不是单一数据点 | 用港股红利指数 + 利率 + 南向资金做代理 |

## 建议的日报判断结构

1. 基础层：每个标的必须有价格、涨跌幅、MA5/MA20、成交额/量能。
2. 映射层：每个标的绑定 2-4 个可量化代理指标，例如：
   - 黄金ETF：金价、美元人民币、美元指数/美债收益率。
   - 紫金矿业：铜价、金价、美元指数、个股趋势。
   - 纳指/标普ETF：纳指/标普、美元人民币、VIX、美债收益率。
   - 港股ETF：恒生科技/恒指、南向资金、港股成交额。
   - 行业ETF：对应行业/概念指数、板块成交额、龙头股趋势。
3. 事件层：新闻/公告只作为加权说明，不能单独给出动作状态。
4. 缺口层：无法自动化的数据必须标为“人工复核”，不伪装成结论。

## 后续实现优先级

1. 扩展美股与利率替代源，补齐纳指、标普500、费城半导体、VIX、美债收益率。
2. 替换不稳定的行业/概念板块接口，优先解决船舶、通信、电网、芯片、创新药。
3. 新闻/公告改为事件标签：政策、订单、财报、风险提示、资金流。
4. 对成交额、折溢价、QDII溢价等指标单独建表，避免混在人工复核项中。
